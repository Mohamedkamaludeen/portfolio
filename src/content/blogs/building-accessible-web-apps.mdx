# Building Accessible Web Applications

Creating accessible web applications isn't just good practice—it's essential. In this guide, we'll explore how to build inclusive digital experiences that work for everyone.

## Why Accessibility Matters

Approximately 15% of the world's population lives with some form of disability. By making your applications accessible, you:

- Reach a wider audience
- Improve SEO and search rankings
- Enhance overall user experience
- Meet legal requirements
- Demonstrate social responsibility

## The WCAG Guidelines

The Web Content Accessibility Guidelines (WCAG) provide a framework with four main principles:

1. **Perceivable**: Information must be presentable to users in ways they can perceive
2. **Operable**: User interface components must be operable
3. **Understandable**: Information and operation must be understandable
4. **Robust**: Content must be robust enough for interpretation by various user agents

## Semantic HTML

Use proper HTML elements for their intended purpose:

```html
<!-- Good -->
<button>Click me</button>
<nav>
  <ul>
    <li><a href="/home">Home</a></li>
  </ul>
</nav>

<!-- Bad -->
<div onclick="handleClick()">Click me</div>
<div class="navigation">...</div>
```

## ARIA Attributes

When semantic HTML isn't enough, use ARIA attributes:

```html
<button aria-label="Close dialog" aria-pressed="false">
  ×
</button>

<div role="alert" aria-live="polite">
  Form submitted successfully!
</div>
```

## Keyboard Navigation

Ensure all functionality is accessible via keyboard:

```tsx
function Modal({ isOpen, onClose, children }) {
  useEffect(() => {
    const handleEscape = (e) => {
      if (e.key === 'Escape') onClose()
    }
    
    if (isOpen) {
      document.addEventListener('keydown', handleEscape)
      return () => document.removeEventListener('keydown', handleEscape)
    }
  }, [isOpen, onClose])
  
  return isOpen ? (
    <div role="dialog" aria-modal="true">
      {children}
    </div>
  ) : null
}
```

## Color Contrast

Maintain sufficient color contrast ratios:

- **Normal text**: Minimum 4.5:1
- **Large text**: Minimum 3:1
- **UI components**: Minimum 3:1

## Focus Management

Make focus indicators visible and logical:

```css
/* Visible focus indicator */
button:focus-visible {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
}

/* Skip to main content link */
.skip-link {
  position: absolute;
  top: -40px;
  left: 0;
}

.skip-link:focus {
  top: 0;
}
```

## Screen Reader Considerations

Provide meaningful content for screen readers:

```tsx
<img 
  src="chart.png" 
  alt="Bar chart showing sales increased by 45% in Q4"
/>

<button aria-label="Add to cart">
  <ShoppingCart aria-hidden="true" />
</button>
```

## Forms and Validation

Create accessible forms with clear labels and error messages:

```tsx
<div>
  <label htmlFor="email">
    Email Address
    <span aria-label="required">*</span>
  </label>
  <input
    id="email"
    type="email"
    aria-required="true"
    aria-invalid={hasError}
    aria-describedby={hasError ? "email-error" : undefined}
  />
  {hasError && (
    <span id="email-error" role="alert">
      Please enter a valid email address
    </span>
  )}
</div>
```

## Testing for Accessibility

Use these tools and techniques:

1. **Automated Testing**: Axe, Lighthouse, WAVE
2. **Keyboard Testing**: Navigate without a mouse
3. **Screen Reader Testing**: NVDA, JAWS, VoiceOver
4. **Manual Review**: Check WCAG compliance
5. **User Testing**: Include users with disabilities

## Common Mistakes to Avoid

- Using div/span for interactive elements
- Missing alternative text for images
- Poor color contrast
- Inaccessible modal dialogs
- Missing form labels
- Keyboard traps
- Moving or flashing content without control

## Accessibility in React

Here's an accessible button component example:

```tsx
interface ButtonProps {
  children: React.ReactNode
  onClick: () => void
  disabled?: boolean
  loading?: boolean
}

function AccessibleButton({ 
  children, 
  onClick, 
  disabled, 
  loading 
}: ButtonProps) {
  return (
    <button
      onClick={onClick}
      disabled={disabled || loading}
      aria-busy={loading}
      aria-disabled={disabled}
    >
      {loading ? (
        <>
          <Spinner aria-hidden="true" />
          <span className="sr-only">Loading...</span>
        </>
      ) : children}
    </button>
  )
}
```

## Conclusion

Building accessible applications benefits everyone. Start by:

1. Using semantic HTML
2. Ensuring keyboard navigation
3. Maintaining color contrast
4. Testing with real users
5. Making accessibility part of your development process

Remember: accessibility is not a feature—it's a fundamental aspect of web development.
